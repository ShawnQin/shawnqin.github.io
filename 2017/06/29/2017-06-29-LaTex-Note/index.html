LaTex 简介

既然我们有强大的 MS Word等文字处理软件，为何还要折腾LaTex？很多情况下Word对于排版真的是力不从心 (至少对我而言)，尤其是当我们写比较复杂的文档时，比如博士论文，或者当文档中有比较多的数学或其他符号时。还记得在Word中插入公式时的那种生不如死的感觉吗？以及你辛辛苦苦把文章写好，老板说你插入的公式他的电脑上不能正确显示的时候。

相比于Word这种“所见即所得“的排版模式，LaTex更像是一种编程语言，它能让你更专注于写作内容本身，而不是花费大量的时间担心各种排版细节。这是我最喜欢LaTex的一个原因。关于排版和LaTex的发展历史，感兴趣的可以参看包太雷所著的《LaTex Note-雷太赫排版系统简介》(这是一本写作风格上十分幽默(逗比)，短小精悍的参考资料，非常值得一读)。以我自己写博士论文的经验来看，花费两天时间调教好博士论文LaTex模板，后续整个写作过程都不用担心诸如目录，图片插入位置、页眉、公式编号、交叉引用、参考文献格式等一些列繁琐的问题。仅仅这一点，就值得我们花点时间来学习并使用，个人强烈推荐用LaTex写博士论文(但唯一不足的是男神汤不使用LaTex，也许最好的办法是把他教会)。
LEX排版相对于所见即所得系统的优势有

  • 高质量，它制作的版面看起来更专业，数学公式尤其赏心悦目
  • 结构化，文档结构清晰
  • 批处理，源文件是文本文件，便于批处理
  • 挂平台，兼容所有的操作系统
  • 开源

不足之处

  • 学习曲线较陡峭，需要一定的学习成本
  • 制作文档过程繁琐，可能需要多次编译，不能实时看到结果
  • 排版样式相对单一，缺乏灵活性
  • 宏包质量良莠不齐

谈到LaTex的历史，就不得不提Donald E. Knuth老爷子(没错，就是那位写The Art of Computer Programming的作者，那套书已经写了四十多年了！)。1962年，24岁的他时候就着手写一本关于编译器的书，结果一发不可收拾，一下写了3000多页，结果出版商说这手稿太厚，最好分卷出版。而当1976年第二卷出版时，Knuth觉得排版是在难看(嗯，强迫症~)，于是一怒之下，决定自己写一个排版的引擎，这就是TEX的由来。1980 年代初期，斯坦福研究所 (Stanford Research Institute, SRI) 的 Leslie Lamport开发了一种新的格式，也就是 LATEX。

对于我们初学者来说，并不用知道诸如引擎、宏包和驱动的概念，这些都被封装在称之为”发行版“(distribution)中了。发行版有时也会带一个编辑器，但未必符合你的口味。常见的发行版如下表

操作系统 发行版 编辑器
通用 TeXLive TeXworks

LaTex的安装

最简单和省心的方式就是直接安装整个发行版，根据自己的操作系统，选择相应的发行版。

Windows

安装LaTex发行版

Windows系统可在此处或清华镜像下载TexLive发行版并安装。

PDF阅读器

常见的阅读器如Adobe PDF reader, Foxit PDF reader, skim等均可。

选择合适的编辑器

这个就是根据个人的喜好了，如果你觉得发行版附带的编辑器不够好，你可以折腾Sublime Text 3之类的编辑器。

Mac OS

建议直接从官网下载并安装MacTex。安装完后，这里面附带安装的有编辑器TeXshop, 一个迷你的编辑器LaTexiT(对于偶尔想在slides中加入数学公式，这是一个非常有用的工具), TexLive utility (用于LaTex宏包的安装和更新) 以及BibDesk (一个小巧的文献管理软件，在tex文档中插入参考文献时非常方便)。

PDF阅读器

建议安装skim(一个非常轻量的pdf编辑器)

编辑器

自带的TeXshop已经足够使用了，如果你像我一样对于”丑陋“无法容忍的话，你可以折腾优雅的Sublime Text 3或者使用TeXstudio

Linux

实验室里的同学中除了我似乎没有人使用linux系统，如果有谁需要在Linux系统下安装的话可以单独问我。
